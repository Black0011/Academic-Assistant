# 知识保护准则

## 受保护文件

以下文件包含跨会话积累的调研知识，禁止直接覆盖（Write 整文件）：

- `data/survey/paper-table.md` — 论文调研总表（核心资产）
- `data/survey/findings.md` — 调研发现文档
- `data/survey/research-state.yaml` — 调研状态
- `data/survey/notes/*.yaml` — 论文阅读笔记
- `data/skills/*.yaml` — 提取的调研策略
- `data/knowledge/cases/*.json` — 调研案例
- `papers/` — 已下载的论文 PDF

## 写操作行为规范

### 必须追加（Append），禁止覆盖（Overwrite）

对受保护文件，默认使用 `StrReplace` 在指定位置追加内容。**绝不使用 `Write` 重写整个文件**。

```
❌ 错误：Write(path="data/survey/paper-table.md", contents="全新内容...")
✅ 正确：StrReplace(path="data/survey/paper-table.md", old_string="定位锚点", new_string="原内容 + 新追加内容")
```

### 新建调研任务时的处理

当用户启动新调研主题时：

1. **先读取**所有受保护文件，了解已有内容
2. **分析用户意图**：是「新开题目替换旧调研」还是「在已有基础上扩展」
3. **如果有歧义，主动询问**："已有 N 篇论文的调研总表，是在此基础上追加新方向，还是归档旧调研后重新开始？"
4. **如需归档**：将旧文件移至 `data/survey/archive/YYYY-MM-DD/` 目录，而非直接删除或覆盖

### 写操作前三步检查

对受保护文件的每次写操作，必须在执行前完成：

1. **Read** — 读取当前文件内容，确认现有数据量（行数、论文数等）
2. **Diff** — 判断本次操作是「追加」还是「替换」，如果新内容比原内容短，触发告警
3. **Confirm** — 如果操作会导致数据丢失（替换/缩短），必须向用户确认

## 自进化触发

当发现以下情况时，Agent 应自动分析是否需要更新规则或 Skill：

| 触发信号 | 分析动作 | 可能的自进化 |
|---------|---------|------------|
| 用户指出数据被覆盖/丢失 | 回溯操作链，找到出错环节 | 强化该文件的保护规则 |
| 新增了受保护文件类型 | 评估是否需要加入保护列表 | 更新本规则的受保护文件列表 |
| 调研流程出现新的最佳实践 | 分析是否可提炼为通用规范 | 更新 autoresearch Skill 或创建新 Skill |
| 重复犯同一类错误 | 分析错误模式 | 创建针对性防御规则 |
