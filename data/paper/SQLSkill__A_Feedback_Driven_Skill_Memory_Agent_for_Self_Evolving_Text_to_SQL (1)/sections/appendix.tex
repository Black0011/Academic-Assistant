\section{Hyperparameter Settings}
\label{sec:hyperparams}

Table~\ref{tab:hyperparams} lists all hyperparameters. Runtime Skill statistics use the following notation: $u_k$ (retrieval count), $G_k$ (cumulative counterfactual gain), $\hat{g}_k = G_k/(u_k + \epsilon_{\text{cf}})$ (normalized gain rate), and $\ell_k^{\min}$ (minimum source evidence level).

\begin{table}[h]
  \centering
  \small
  \caption{Complete hyperparameter listing for SQLSkill.}
  \label{tab:hyperparams}
  \begin{tabular}{llrl}
    \toprule
    \textbf{Symbol} & \textbf{Meaning} & \textbf{Default} & \textbf{Category} \\
    \midrule
    \multicolumn{4}{l}{\emph{Core method parameters (discussed in main text)}} \\
    $N_{\text{recur}}$ & Recurrence count for $L_0 {\to} L_1$ & 3 & Evidence \\

\section{Error Signature Example}
\label{sec:error_sig_example}

Below is an example of the error archiving output for a failed query ``calculate monthly sales growth rate per product category'' on an e-commerce database:

\begin{lstlisting}[language=,caption={Example error archiving output.}]
error_vector: [0, 1, 1, 0]
  # Field activated, Relation activated
primary_dimension: Relation

z_field(q):
  pattern_label: wrong_measure_field
  schema_element: orders.total_amount
    (should be order_items.unit_price * qty)
  ast_feature: SELECT_aggregate_on_wrong_column

z_relation(q):
  pattern_label: missing_join
  schema_element: orders -> products
  ast_feature: no_intermediate_table

diagnostic_tags:
  [aggregation_error, missing_bridge_table]
root_cause: "Directly aggregated on orders table,
  missing order_items bridge table, causing
  wrong measure field and incomplete JOIN path"
\end{lstlisting}

This Case is simultaneously counted toward both the $(D_{\text{field}}, z_{\text{field}})$ and $(D_{\text{relation}}, z_{\text{relation}})$ cluster counters for recurrence detection.
